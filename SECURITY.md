# Security Policy

## Supported Versions

Security fixes target the current `main` branch until the project starts
publishing versioned releases.

## Reporting a Vulnerability

Please do not report security vulnerabilities in a public issue.

Use GitHub's private vulnerability reporting for this repository when
available. If that is unavailable, contact a maintainer through a private
channel and include:

- affected version or commit
- steps to reproduce
- expected impact
- whether the issue is already public or being actively exploited

We will acknowledge reports as soon as practical, investigate privately, and
coordinate disclosure after a fix or mitigation is available.
